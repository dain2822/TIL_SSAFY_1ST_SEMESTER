### **변수**(Variable)

: 값을 나중에 다시 사용하기 위해 그 값에 붙여주는 고유한 이름

**변수 할당**(Variable Assignment)

: 표현식이 만들어 낸 값에 이름을 붙이는 과정(연결)

**할당문**(Assignment Statement)

: “값 36.5를 변수 degrees에 할당했다.”

→ `degrees = 36.5`

- degrees : 변수 이름
- = : 할당 연산자(오른쪽 표현식의 평가 결과 값을 왼쪽 변수에 저장)
- 36.5 : 표현식

→ ‘=’는 수학에서의 ‘같다’가 아니라 오른쪽 값을 왼쪽 변수에 ‘할당’한다는 뜻

**변수명 규칙**

- 영문 알파벳, 언더스코어(_), 숫자로 구성
- 숫자로 시작 불가
- 대소문자 구분
- 파이썬 내부 예약어 사용 불가

![image.png](attachment:774d0926-ce63-497d-b8eb-2aced963d305:image.png)

**변수, 값 그리고 메모리**

: 거리에 집 주소가 있듯이 메모리의 모든 위치에는 그 위치를 고유하게 식별하는 **메모리 주소**가 존재

![image.png](attachment:16667302-11d2-4a64-ac3f-6e69d114b415:image.png)

*메모리 주소: 컴퓨터가 특정 데이터 값을 정확히 찾아가기 위해 사용하는 기계적인 숫자 주소

- 고유한 ID(메모리 주소) → 제품의 바코드
- 타입(Type) → 제품의 종류(예: 정수, 실수, 문자열 등)
- 값(Value) → 제품의 실제 내용물
- 값+타입+주소 정보를 묶은 것을 객체(Object)라고 부름

: 변수는 특정 객체를 ‘가리키는(refer/point to)’ 이름표

→ 변수는 메모리 주소를 ‘가지지(contain)’ 않음

![image.png](attachment:35a931da-be45-458a-8985-4c972d2d3642:image.png)

**변수** ⇒ 값을 나중에 다시 사용하기 위해 그 값에 붙여주는 고유한 이름 / **객체를 가리키는 이름**

**할당문 동작 순서**

![image.png](attachment:ad5f9648-19fe-4cbe-ac35-18b7a8b535a5:image.png)

1. 오른쪽 표현식 평가

→ 할당 연산자(=)의 오른쪽에 있는 표현식 전체를 계산하여 하나의 결과값(객체)를 만듦

1. 왼쪽 변수명 확인

→ 이름이 처음 사용되었다면 새로운 ‘이름표’ 준비

→ 이미 존재하는 이름이라면 기존 ‘이름표’를 그대로 사용

1. 변수명과 결과값 연결(참조)

→ 왼쪽의 변수명이 오른쪽에서 만들어진 결과값을 가리키도록 연결

→ 변수명이 이전에 다른 객체를 가리키고 있었다면 그 연결은 끊어지고 새로운 객체와의 연결만 남음(**재할당**)

**재할당(Reassignement)**

: 이미 값이 할당된 변수에 새로운 값을 다시 할당하는 

: 변수는 특정 값을 ‘기억’하거나 ‘가리키는’ 이름, 재할당은 이 변수가 가리키는 대상을 새로운 값으로 변경하는 행위

![image.png](attachment:5d2fdd39-1a44-481e-a50f-598fbac03acd:image.png)

- 재할당이 일어나면 해당 변수의 id값도 변경됨(`id(변수명)`을 통해 출력 가능)

### **Sequence Types**

: 여러 개의 값들을 **순서**대로 **나열**하여 저장하는 자료형

*시퀀스: 여러 데이터가 정해진 순서대로 일렬로 늘어선 자료 구조

*파이썬의 대표 시퀀스 타입: str, list, tuple, range

*인덱스(Index): 시퀀스 자료형에서 각 값의 위치를 식별하기 위해 부여된 고유 번호

- 각 데이터는 기차 칸처럼 순서대로 연결되어 있음
- 모든 칸에는 0번부터 시작하는 고유한 번호(인덱스)가 붙어 있음
- 이 번호를 통해 원하는 칸의 데이터에 바로 접근할 수 있음

**시퀀스 타입의 5가지 공통 특징**

1. 순서(Order) : 값들이 순서대로 저장(정렬X)
2. 인덱싱(Indexing) : 각 값에 고유 번호(인덱스)를 가지고 있으며, 인덱스를 사용하여 특정 위치의 값을 선택하거나 수정할 수 있음
3. 슬라이싱(Slicing) : 인덱스 범위를 조절해 전체 데이터 중 원하는 부분만 값을 잘라내서 사용할 수 있음
4. 길이(Length) : len() 함수를 사용하여 저장된 값의 개수(길이)를 구할 수 있음
5. 반복(Iteration) : 반복문을 사용하여 각 값을 하나씩 순서대로 꺼내 사용할 수 있음
